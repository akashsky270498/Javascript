Q:5 Explain Memory Management & Potential Memory Leaks in Nodejs?
Ans: Nodejs manages memory automatically using the garbage collector in the v8 engine, which uses a mark & sweep algorithm approach to remove the unused objects. However, memory leaks can still happen if refrences are unintentionally kept in the application.

- Common causes of Memory Leaks are:
- Unclosed browser instances in Puppeteer.
- Unresolved Promise holding closures.
- Accumulating Event Listeners.
- Large Unbounded Caching.
- We should also avoid Global Variables & Timers and Intervals.

For Debugging Memory Leaks I use:
- I run nodejs with --inspect flag to take the Heap snapshots in Chrome DevTools.
- We also use clinic.js, especially tools like  Doctor & Heap Profiler to analyze Memory Usage & detect performance bottlenecks in Nodejs applications.

Q:6 Exaplin How JS is Single Threaded? What is Event Loop & How does it work?
Ans: Javascript is single threaded which means it can execute one task at a time using the callstack. To handle the async tasks like Timers, API calls or file operations JS make uses of event loop.

- Event loop is a mechanism in JS which is use to handle the async operations, instead of waiting for the longer taks like Timers, API calls or file operation JS make use of an event loop which handle this tasks on the background while contiue executing the other parts of the code, this make JS efficient & non-blocking.

- CallStack is where the Sync code runs. Callstack is the collection of functions which are under execution, if no functiona are are there Global Anonymous function are present once it finishes the sync code execution it picks the tasks from Microtask Queue (Promises) & then from Macrotask Quere(Set time functions). Microtask Queue have > priority then Macrotask Queue.

ex: 
setTimeout(() => console.log("timeout"), 0);
Promise.resolve().then(() => console.log("promise"));

Q:7 Difference Between Promise.all, Promise.allSettled, Promise.race, Promise.any?
Ans: 
- Promise.all(): Waits for all promise to get succeed, If one fails, all promise fails immediately.
Usecase: when u need all results together.

- Promise.allSettled(): Waits for all promises to get finished, Doesn't fail if any of the promise get rejected, return status + value/reason.
Usecase: When u want results even if some fails.

- Promise.race(): Returns results of first promise that settled (resolve or rejected).
Usecase: Timeout pattern.

- Promise.any(): Return the first successful promise, Ignores rejected one, Fails only if all promises rejected.
Usecase: Fallback strategy.
Ex: Try multiple APIs return whichever works fine.
