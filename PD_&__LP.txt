Q: Explain your project?
Ans: 
- I have worked on Talentprise AI based talent marketplace handlig 50k+ DAU.
- It is a microservices based backend built on NestJS, Postgres, Redis & AWS.
- I was responsible for building scalable APIs, Optimizing database Queries & Improving performnces using redis chaching.
- One key improvement I made was improving the API latency by 30% by introducing cahing & optimizing queries.
- We also ensured high availbility using proper error handling, retries, timeout & monitoring.  

Q: How did you design your APIs?
Ans: I design my APIs using REST std principle clear response based endpoints, HTTP Methods & Status Code.
I also keep APIs stateless & use validation at request level. For Complex use cases I use GraphQL, for flexible data fetching.

Q: How did you handle versioning?
Ans: I used URI like /api  /v1/. This helps backend compatiblility.

Q: Why PostgreSQL?
Ans:  PostgreSQL bcoz it is realible, ACID Compliant & can handle complex queries as well. It is also suitable for realtional data, supports indexing, JOINS and ensure data consistency.

Q: Hou did you optimized the query?
Ans: I optimized query by adding necessary & appropriate indexing, removing unnecessary joins, selected required fields & used pagination for large date sets. I also analyzed slow queries & improved them.

Q: What indexing did you used?
Ans: I used both B-Tree indexing & Composite indexing, B-Tree indexing for column used in WHERE, JOINS & ORDER BY & Composite indexes for frequently queried column combinations.

Q: Where exactly did you used caching?
Ans: I used Redis caching for frequently accessed data like user profile & search results to reduce database loads & to improve response time.

Q: Which chaching strategy u used?
Ans: I used cache-aside strategy, first check the Redis, If data not found fetch it from database and then return.

Q: How JWT Works?
Ans: JWT contains encoded user details which is sent with every API request from the client side through Headers & server verifies it to authenticate the requests without storing session.

Q: How do you secure APIs?
Ans: Using JWT Authentication, RBAC, CORS, User Validation & Sanitization, Rate Limiting & Throttling, Data Encryption at REST & Also ensuring sensitive data should not get exposed.

Q: Why Microservices over Monolithic?
Ans: Microservices over Monolithic bcoz first reason is Independent Scaling, second is Better Fault Isolation & the thrid one is Faster Development since services are loosely coupled.

Q: How Services Communicate?
Ans: Services can communicate mainly through REST APIs. For async communication we can make use of Messaging Queues like RabbitMQ as well but in my project we have used gRPC.

Q: What happens if Redis goes down?
Ans: If Redis goes down system falls back to database. Although it may increase Latency, but our priority should be system should still work. Later on we use retry mechanism or monitor to recover quickly.

Q: How do you handle High Traffic?
Ans: I handled High Traffic using Redis(caching), Load Balancing, Horizontal Scaling & Optimizing Database Queries. Also we can use Rate Limiting to prevent abuse.

Q: How do you ensure 99.5% Uptime?
Ans: I ensured 99.5% uptime by designing Fault Tolerence System using Retries, Timeout, Better Logging & monitoring, Proper Error Handling & Avoiding Single Point of Failure.

Q: How do you debug Production Issue?
Ans: I debug production issue using following techniques:
- I will jump to Logging & monitoring metrices.
- I will go through the failing services deeply & try to discover the cause of failure.
- I will copy the curl from production env & test it in my local.
- I will check for the dependencies.
- I will try to rollback to the previous commit.
- I will do hotfix.
