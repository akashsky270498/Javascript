Q: Tell me about your experience with AWS?
Ans: I have primarily worked on AWS backend-focused role where I handled Storage, Performance Opimization, Deployments of the Applications and Monitoring.
In my projects I have used S3 for storing User generated content like Images & Documents. To improve the Performance & reduce Latency, I integrated CloudFront as a CDN layer on the top of S3.
For backend services I have used EC2 to deploy the NodeJS applications, manage runtime environments & handle production workloads.
On the Monitoring side I have used CloudWatch extensivley for logging, debugging issues & tracking system performance.
I also have hands on exposure to AWS Lambda, where I built event driven functions for asynchronous processing, helping reduce API latency & decouple services.

Q: Where you have used S3?
Ans: 
- I have used S3 for storing the user-uploaded Images & Documents.
- I have also used for pre-signed URLs for secure uploads.
- Avoided storing files on backend servers (better scalability).

How?
- Backend generates pre-signed URLs using AWS SDK.
- Client uploads directly on S3 Bucket.
- Reduces Load on Backend Servers & improves performance.

Q: Why CloudFront?
Ans:
- To reduce Latency by caching the content closer to the user.
- Reduces load on S3 bucket as well.
- Imrpove UX.

How?
- Created CloundFront Distribution.
- Connected it with S3 bucket.
- Used CDN URLs insted of direct S3 URL.

Q: Explain you system architecture?
Ans:
- Client -> hits backend API (EC2/NestJS service).
- API handles the request + validation.
- For Sync Work - DB(Postgres/MySQL).
- For async Work - trigger Lambda (via SDK).
- Lambda process Background tasks like notifications.
- Static assests - S3.
- CDN - Cloudfront.
- Logs/metrices - Cloudwatch.

Key design points
- Stateless backend Services - Easy Scaling.
- Redis Caching - Reduces DB load.
- Async Processing - Better Latency.
- Observability via Logs & Metrices.
