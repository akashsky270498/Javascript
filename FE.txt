Q: How did redis reduce response time by 30%?
Ans: We observed that some endpoints like user profile, following/followers & frequently accessed lists were being queried from the database on every request. So I implemented redis caching for those endpoints using a cache aside strategy. When a request comes we check the redis first if the data is present we return it directly else we fetch data from database and cache it in redis. This reduce the database load and improve the response time significantly.

Q: What happens inside nodejs when an API request comes?
Ans: When a request hits the server, Nodejs receives it through the HTTP server modules. The request is placed in the event loop queue. If the request requires I/O operations like database queries or external API calls, those operations are delegated to the Node.js worker threads through libuv. While waiting for the response, the event loop continues processing other requests. Once the I/O operation completes, the callback or promise is placed back in the event loop and the response is returned to the client.

======================================================================================================================================
Q: Why is nodejs good for high concurrency applications?
Ans: Nodejs is good for high concurrency applications bcoz of it Non-Blocking I/O event driven architecture. Instead of creating thread for each & every request like traditional servers. Nodejs uses a single threaded event loop to handle all the request async.

Q: How would you design a scalable authentication system?
Ans: A scalable authentication system typically includes JWT-based authentication, secure password hashing, refresh tokens, and centralized user management. Passwords are stored using hashing algorithms like bcrypt or argon2. Access tokens are used for API authentication, and refresh tokens allow issuing new access tokens without requiring the user to log in again. For large systems, session storage or token revocation lists may be managed using Redis.
